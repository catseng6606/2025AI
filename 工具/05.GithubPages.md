# 使用 GitHub Pages

利用 GitHub Pages 建立免費的靜態網站託管服務。

## 方法一：使用 GitHub 網頁介面

1.  **登入** GitHub。
2.  **建立新專案**：
      - 點選右上角的 `+` 號，選擇 `New repository`。
      - 或進入個人首頁的 `Repositories` 分頁，點選 `New`。
3.  **設定 Repository 資訊**：
      - `Repository name`：輸入專案名稱（例如：`hello`）。
      - `Public/Private`：確認勾選 `Public`（免費版 GitHub Pages 需設為公開）。
      - 點選 `Create repository`。
4.  **上傳網頁檔案**：
      - 在新建立的頁面中，點選 `uploading an existing file` 連結。
      - 拖曳或選擇你的 `index.html` 檔案。
      - 在下方 `Commit changes` 區塊，直接點選綠色的 `Commit changes` 按鈕（這會自動建立 `main` 分支）。
5.  **開啟 GitHub Pages 功能**：
      - 點選該 Repository 上方的 `Settings` 分頁。
      - 在左側選單點選 `Pages`。
      - 在 **Build and deployment** 區塊下的 **Branch** 選項，將 `None` 改為 `main`，並按下 `Save`。
6.  **取得網站連結**：
      - 等待約 1-2 分鐘後重整頁面，頂部會顯示 `Your site is live at...`。
      - 網址格式通常為：`https://<你的帳號>.github.io/<專案名稱>/`

## 方法二：使用終端機指令 (Terminal)

此方法適合已安裝 Git 工具的使用者。

1.  **準備專案與上傳** (以 PowerShell 為例)：
    在本機建立資料夾與 `index.html`，並推送到 GitHub。

    ```powershell
    # 1. 建立資料夾並進入
    mkdir hello
    cd hello

    # 2. 建立 index.html (簡單範例)
    echo "<!doctype html><html><head><meta charset='utf-8'><title>Hello</title></head><body><h1>Hello GitHub Pages</h1></body></html>" > index.html

    # 3. 初始化 Git 並提交
    git init
    git add index.html
    git commit -m "Initial commit"
    git branch -M main

    # 4. 連結遠端 Repository (請將 {user} 替換為你的帳號)
    # 注意：需先在 GitHub 網頁上建立一個空的 Repository 名為 hello
    git remote add origin https://github.com/{user}/hello.git

    # 5. 推送至 GitHub
    git push -u origin main
    ```

2.  **設定 GitHub Pages**：

      - 推送成功後，參照「方法一」的步驟 5 與 6，進入 `Settings` \> `Pages` 啟用服務。
